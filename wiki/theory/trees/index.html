Trees

Theory Hub · Binary Trees · BST · Traversal · LCA

A tree is a connected, acyclic graph: a set of nodes linked by edges where exactly one node (the root) has no parent, and every other node has exactly one parent. Most interview trees are binary — each node has at most a left and a right child — and the recurring move is recursion that mirrors the structure: to solve a problem on a node, solve it on its children and combine. Almost every tree problem is "do a traversal, and decide what to compute on the way down (pre-order), the way up (post-order), or in sorted order (in-order)."

This hub gives you the vocabulary first — tree, binary tree, BST, trie, and the traversal orders — then sends you to the deep dives for specific families. The first deep dive is Lowest Common Ancestor (LCA), the canonical "find the deepest node that is an ancestor of both targets" problem, which underpins a large share of Airbnb's tree questions.

Vocabulary you must know cold
TermPlain meaningExample / note
TreeA connected graph with no cycles; n nodes have exactly n−1 edgesOne root, every other node has a unique parent
Node / edgeA node holds a value; an edge links a parent to a childThe basic building blocks
RootThe single node with no parent — the entry point to the whole treeTraversals start here
LeafA node with no childrenThe "bottom" of each branch
Parent / childIf an edge goes from A down to B, A is B's parent and B is A's childEach node (except root) has exactly one parent
Ancestor / descendantAn ancestor is any node on the path up to the root; a descendant is any node in your subtreeThe basis of LCA — a node is its own ancestor by convention here
SubtreeA node together with all of its descendantsRecursion solves a subtree, then combines
Binary treeEvery node has at most two children, named left and rightThe default interview tree
Binary search tree (BST)A binary tree where, for every node, all left-subtree values are smaller and all right-subtree values are largerIn-order traversal yields sorted values; lookup is O(height)
Balanced treeLeft and right subtree heights differ by at most a constant at every node, so height is O(log n)What LC 108 builds from a sorted array
Height / depthHeight = longest root-to-leaf edge count; depth of a node = edges from the root to itA balanced tree of n nodes has height ≈ log₂ n
Trie (prefix tree)A tree whose edges are labeled by characters; each root-to-node path spells a string prefixNot a binary tree — used for word/prefix lookup (e.g. autocomplete, LC 336)
Pre-orderVisit the node, then left subtree, then right (node-first)Good for copying/building a tree top-down
In-orderLeft subtree, then node, then right (node-in-the-middle)On a BST, produces values in sorted order
Post-orderLeft subtree, then right, then the node (node-last)Good for combining children's results — the shape LCA uses
Level-order (BFS)Visit nodes layer by layer from the root, using a queueShortest-path-in-edges, level grouping
The mental default: reach for recursion that mirrors the tree — handle the base case (usually nil), recurse on left and right, and combine. Choose when you do the node's own work: pre-order (before recursing) to push information down, post-order (after recursing) to pull results up. LCA is the archetypal post-order problem.
