Graphs

Theory Hub · BFS · Topological Sort · Dijkstra

A graph is a set of vertices (nodes) connected by edges (links). Almost every "shortest path", "reachability", "ordering", or "spread" question is secretly a graph problem — the hard part is usually recognizing the graph, not traversing it. The grid in a maze, the dependency between two courses, the one-bit difference between two board states: each becomes vertices and edges, and then one of a small number of standard traversals answers the question.

This hub gives you the vocabulary first — vertex, edge, neighbor, the adjacency representations, and the three traversal families — then sends you to the deep dives. The first deep dive is Breadth-First Search (BFS), the level-by-level traversal that finds the fewest-steps path in an unweighted graph. Topological Sort and Dijkstra follow.

Vocabulary you must know cold
TermPlain meaningExample / note
Vertex (node)One entity in the graphA cell in a grid, a course, a board configuration
EdgeA connection between two vertices"you can step from A to B" / "A must come before B"
Neighbor (adjacent)A vertex reachable in one edge from the current oneThe four (or eight) cells around a grid cell
Directed vs undirectedDirected edges go one way (A→B); undirected go bothCourse prerequisites are directed; a maze's steps are usually undirected
Weighted vs unweightedWeighted edges carry a cost; unweighted edges all "cost 1"BFS solves unweighted shortest path; Dijkstra solves weighted
DegreeHow many edges touch a vertex (in-degree / out-degree if directed)Kahn's topo sort starts from in-degree-0 vertices
PathA sequence of vertices each joined to the next by an edge"shortest path" = fewest edges (BFS) or least total weight (Dijkstra)
CycleA path that returns to its startA directed cycle makes a topological order impossible
Connected componentA maximal set of vertices all reachable from one another"how many islands" counts components
DAGDirected Acyclic Graph — directed edges, no cycleThe exact precondition for a topological ordering
Adjacency list[vertex: [neighbors]] — store each vertex's neighborsThe default representation; O(V+E) space
Adjacency matrixA V×V grid where m[i][j]=1 means an edge i→jO(V²) space; good for dense graphs or fixed small V
Implicit graphVertices/edges computed on the fly, never storedGrid neighbors, or "states one move apart" — the BFS-over-states pattern
Visited setThe vertices already enqueued/processed, to avoid revisitingThe single most-forgotten line in any BFS/DFS
BFS (breadth-first)Explore level by level with a queueFewest-edges shortest path in an unweighted graph
DFS (depth-first)Go as deep as possible before backtracking, with a stack/recursionConnectivity, cycle detection, topo sort
The mental default: first ask "what is a vertex, what is an edge?" — even for a grid or a set of strings. Then pick the traversal by the question: fewest steps in an unweighted graph → BFS; any reachability / connectivity / cycle → DFS or BFS; valid ordering under dependencies → topological sort; cheapest path with edge weights → Dijkstra. BFS is the archetype this hub starts with.
