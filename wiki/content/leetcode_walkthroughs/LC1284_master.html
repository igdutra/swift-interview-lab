Theory
bfs_deep_dive_master.html

Given an m×n binary matrix mat, one step chooses any cell and flips it together with its four direct neighbors (up, down, left, right — diagonals excluded, out-of-grid neighbors ignored). Return the minimum number of steps to turn the whole matrix into all zeros, or −1 if it's impossible. The grid is tiny: m, n ≤ 3, so there are at most 9 cells.

InputOutputWhy
[[0,0],[0,1]]3Three flips (each toggling a plus-shape) clear it; no shorter sequence exists
[[0]]0Already all zeros — zero steps
[[1,0,0],[1,0,0]]-1No sequence of plus-flips reaches the zero matrix

The twist is realizing this is a shortest path between configurations, not a grid-traversal. A "vertex" is the entire matrix state, and an "edge" is applying one flip. Because the grid is so small (≤ 9 cells), the whole matrix fits in one Int bitmask, the visited set becomes a set of integers, and "fewest steps" is exactly BFS over those states — the first time BFS reaches state 0, that layer is the answer.

2   Reading the Problem — insights from the constraints

"Minimum number of steps": "minimum number of moves" where each move costs the same is the BFS signal — but the moves here aren't grid steps, they're transformations of the whole matrix. So the graph is a state space: each reachable matrix is a vertex, each flip is an edge. BFS's first arrival at the zero matrix is the fewest flips.
"m, n ≤ 3" — the constraint that picks the technique: at most 9 cells means at most 2⁹ = 512 distinct matrix states. A space that small is fully searchable, and 9 bits fit in a single Int. This tiny bound is what makes "treat the whole matrix as one vertex" tractable — with a 30×30 grid you couldn't.
"Flip a cell and its four direct neighbors": a flip is a fixed XOR. For each cell position, precompute a mask with bits set for that cell plus its in-grid up/down/left/right neighbors. Applying a flip is state ^ flipMask[cell] — toggling exactly those bits. There are only m·n possible flips (one per cell), so each state has ≤ 9 outgoing edges.
"Or −1 if impossible": some start matrices can never reach all-zeros (the flip operations span only a subspace). BFS handles this for free — if the queue drains without ever dequeuing state 0, return −1. No special reachability theory needed.
Encoding as bit position: cell (r, c) maps to bit r·n + c (row-major). The whole matrix becomes one integer by OR-ing 1 << (r·n + c) for every 1-cell. The zero matrix is integer 0 — a clean goal test.

The "already zero" and "unreachable" cases bracket the search. Check the start against the goal first so an all-zero input returns 0 without work. At the other end, never assume a solution exists — let BFS prove unreachability by draining the queue. The XOR-is-its-own-inverse fact means re-applying a flip undoes it, so the visited set is what stops the search from looping.

4   Try It Yourself First

The leap is "the whole matrix is one vertex." Open hints one at a time.

Hint 1 — the constraint that's a clue

m, n ≤ 3 means at most 9 cells and at most 2⁹ = 512 whole-matrix configurations. A space that small is fully searchable. What if each entire matrix were a single node in a graph?

Hint 2 — the pattern

"Minimum number of moves" + equal-cost moves = BFS. The vertex is a matrix state, the edge is "apply one flip." Encode each matrix as an Int bitmask (cell r,c → bit r·n+c). BFS from the start integer to the integer 0; the layer you first reach 0 is the answer.

Hint 3 — the flip as an XOR

For each cell, precompute a mask with bits for that cell and its in-grid up/down/left/right neighbors. Applying the flip is state ^ mask — XOR toggles exactly those bits. Keep a visited set of seen states so you never reprocess one; if the queue empties without reaching 0, return −1.

5   Interview Simulation

The interviewer has stated the problem. The script starts with your first words — practice it out loud.

Step 1 — Clarifying questions
Q: "How large can the matrix be?"
A: m and n are both at most 3. → At most 9 cells, ≤ 512 states — small enough to encode the whole matrix in one Int and search the entire state space.
Q: "A flip toggles the cell and which neighbors — diagonals too?"
A: Only the four direct (orthogonal) neighbors, and out-of-grid ones are ignored. → One precomputed XOR mask per cell, only in-grid neighbor bits set.
Q: "Is it guaranteed solvable, or can it be impossible?"
A: It can be impossible. → I return −1 if BFS drains without reaching the zero state.
Q: "Does flipping the same cell twice ever help?"
A: No — XOR is its own inverse, so two flips cancel. → A visited set of states prevents loops and redundant work.
Step 2 — The plan, before any code
The plan, spoken

"This is state-space BFS — Variant 8.3 of the Graphs / BFS block. The signal is 'minimum number of moves,' and the tiny m, n ≤ 3 bound is the tell that I should treat the entire matrix as a single vertex. The graph is implicit: vertices are matrix configurations, edges are flips."

"Let me pin the conditions before the structures. First, the encoding: cell (r,c) is bit r·n+c, so the whole matrix is one Int, and the goal is the integer 0. Second, the edge: for each cell I precompute a flip-mask with bits for that cell and its in-grid orthogonal neighbors; applying a flip is state ^ mask. The validity condition for enqueuing a next state: it hasn't been visited. The invariant: each state enters the queue once because I mark it visited on enqueue — that bounds the work at the 512 states. The stop condition: I return the current layer the moment I dequeue state 0; if the queue empties first, the target is unreachable → −1."

"On structures: an Int for the start state (built by OR-ing bits over the 1-cells); an array of m·n precomputed flip-masks; a queue of states with a head index; a Set<Int> of visited states. A per-layer step counter via the level-size loop."

"The walk: encode the start matrix. If it's already 0, return 0. BFS layer by layer: for each state, try all m·n flips, XOR to get the next state, enqueue the unvisited ones. When I dequeue 0, return the layer count. Time is O(states · m·n) = O(2^(m·n)·m·n), bounded by 512·9; space O(states) for visited."

6   Why This Approach — and not the others

CandidateCostVerdict
Brute-force all flip subsetsO(2^(m·n) · m·n)Each cell is flipped 0 or 1 times (twice cancels), so 2^(m·n) subsets — same order as BFS, but you'd still need to find the minimum subset, i.e. count set bits. Works, but BFS finds the minimum naturally by layer.
DFS / backtracking over flip sequencesexponential, no min guaranteeWrong tool. DFS explores one deep sequence first and has no shortest-path property; you'd have to exhaust all sequences to be sure of the minimum.
Gaussian elimination over GF(2)O((m·n)³)The "proper" linear-algebra solution (this is a Lights-Out problem) — elegant and scales, but heavy to derive in an interview. Overkill given m, n ≤ 3.
State-space BFS over bitmasksO(2^(m·n) · m·n)Chosen. The small state space makes full BFS trivial, the minimum falls out of the layer count, and the encoding + XOR-flip is clean and fast to write correctly.
The deciding signal: "minimum moves to transform a configuration" + a tiny state space. That pair says "make each configuration a vertex and BFS." The ≤ 9-cell bound is the green light to encode the whole matrix as one integer; without it, BFS over states would blow up and you'd need the GF(2) approach.

7   Data Structures — choice and justification

StructureHoldsWhy this and not weaker/stronger
Int bitmask (per state)The whole matrix encoded — bit r·n+c = cell value9 bits fit in one Int, making a matrix a hashable, comparable scalar. The goal is just 0. A 2-D array per state would be unhashable and slow to compare.
flipMasks: [Int] (one per cell)For each cell, the bits its flip toggles (itself + in-grid neighbors)Precomputing once turns each edge into a single state ^ mask XOR — O(1) per move instead of re-deriving neighbors every time.
queue: [Int] + headThe frontier of states, FIFOFIFO order makes first-arrival = fewest flips. Head index keeps dequeue O(1); removeFirst() would add an O(states) factor.
visited: Set<Int>States already enqueuedO(1) membership so each of the ≤ 512 states is processed once. Keying on the Int state is the whole reason the encoding matters.
steps counter (per layer)Flips applied so farThe level-size loop bumps it once per BFS layer = once per flip-depth.
Why the bitmask encoding is the crux. The entire problem becomes easy because a matrix is an Int: the visited set is a Set<Int>, the goal is 0, equality is integer equality, and a flip is one XOR. Trying to BFS over [[Int]] states directly would force you to make 2-D arrays hashable and copy them constantly. The encoding is the technique; the BFS loop is boilerplate.

8   Pseudocode Dry Run

Pseudocodeencode → BFS over states → first arrival at 0
n ← cols
start ← OR of (1 << (r*n + c)) for every cell with mat[r][c] == 1
FOR each cell (r,c): flipMask[r*n+c] ← bits for (r,c) + in-grid 4-neighbors

IF start == 0: RETURN 0          already the zero matrix
queue ← [start] ; visited ← { start } ; steps ← 0
WHILE queue not empty:
    levelSize ← size(queue) ; steps ← steps + 1
    REPEAT levelSize times:
        state ← dequeue(queue)
        FOR mask IN flipMask:
            next ← state XOR mask
            IF next == 0: RETURN steps     first arrival = fewest flips
            IF next not in visited:
                visited.insert(next) ; enqueue(queue, next)
RETURN -1                          unreachable

Trace on [[0,0],[0,1]] (2×2, n=2) — expected 3. Bit layout: bit0=(0,0) bit1=(0,1) bit2=(1,0) bit3=(1,1), so start = 1<<3 = 8 = 0b1000.

Traceflip-masks then the first BFS layers
flip masks (cell + orthogonal in-grid neighbors), 2×2:
  cell0 (0,0): self + right(1) + down(2)        = 0b0111 = 7
  cell1 (0,1): self + left(0) + down(3)         = 0b1011 = 11
  cell2 (1,0): self + up(0)  + right(3)         = 0b1101 = 13
  cell3 (1,1): self + up(1)  + left(2)          = 0b1110 = 14
start = 0b1000 = 8

steps=1, expand 8:
  8^7  = 15 → enqueue ; 8^11 = 3  → enqueue
  8^13 = 5  → enqueue ; 8^14 = 6  → enqueue   (none is 0)
steps=2, expand {15,3,5,6}:
  e.g. 3^7=4, 3^11=8(seen), 3^13=14, 3^14=13, 15^… , ...

  one flip from some layer-2 state yields next == 0 → RETURN 3

The exact layer-2/3 states are many; the point is that state 0 is first produced as an XOR result at depth 3 — so the minimum is 3 flips.

9   Swift Solution — iterative

Encode the matrix into an Int, precompute one flip-mask per cell, then BFS over integer states with a level-size loop for the step count.

A.1 State-space BFS over a bitmask
O(2^(m·n)·m·n)O(2^(m·n))
Plan
Encode the matrix as an Int (cell r,c → bit r·n+c). Precompute, for each cell, the XOR mask its flip toggles (the cell plus its in-grid orthogonal neighbors). If the start is already 0, return 0. BFS layer by layer: for each state, XOR with every flip-mask to get next states; the first time a next state is 0, return the current layer. Empty queue → −1. 
func minFlips(_ mat: [[Int]]) -> Int {
    let rows = mat.count, cols = mat[0].count

    // 1. Encode the matrix into a single Int: cell (r,c) → bit (r*cols + c).
    var start = 0
    for r in 0..<rows {
        for c in 0..<cols where mat[r][c] == 1 {
            start |= 1 << (r * cols + c)
        }
    }
    if start == 0 { return 0 }                       // already the zero matrix

    // 2. Precompute one flip-mask per cell: itself + in-grid orthogonal neighbors.
    let directions = [(0,0),(0,1),(0,-1),(1,0),(-1,0)]
    var flipMasks: [Int] = []
    for r in 0..<rows {
        for c in 0..<cols {
            var mask = 0
            for (dr, dc) in directions {
                let nr = r + dr, nc = c + dc
                if nr >= 0, nr < rows, nc >= 0, nc < cols {
                    mask |= 1 << (nr * cols + nc)
                }
            }
            flipMasks.append(mask)
        }
    }

    // 3. BFS over integer states; the layer that first reaches 0 is the answer.
    var queue = [start], head = 0, steps = 0
    var visited: Set<Int> = [start]
    while head < queue.count {
        let levelSize = queue.count - head
        steps += 1
        for _ in 0..<levelSize {
            let state = queue[head]; head += 1
            for mask in flipMasks {
                let next = state ^ mask                  // apply one flip
                if next == 0 { return steps }            // first arrival = fewest flips
                if !visited.contains(next) {
                    visited.insert(next)                  // mark on enqueue
                    queue.append(next)
                }
            }
        }
    }
    return -1                                          // zero matrix unreachable
}
Note: the direction table includes (0,0) so the cell itself is part of its own flip-mask. Bounds are checked while building each mask, so out-of-grid neighbors are silently skipped — corner cells get smaller masks automatically. The steps += 1 sits at the top of each layer, so a next-state of 0 found while expanding a layer-(k−1) state returns k.

12   Complexity

PhaseTimeSpaceWhy
Encode + build flip-masksO(m·n)O(m·n)One pass over the cells to encode; one pass per cell over ≤ 5 directions to build masks.
BFS over statesO(2^(m·n) · m·n)O(2^(m·n))At most 2^(m·n) states, each enqueued once; expanding a state tries all m·n flips. Visited holds up to 2^(m·n) integers.
TotalO(2^(m·n) · m·n)O(2^(m·n))The state count dominates — exponential in the cell count, but m·n ≤ 9 caps it at 512·9 ≈ 4.6k operations. The bottleneck is the size of the state space, not the BFS loop.
Why the exponential is fine here. 2^(m·n) looks alarming, but the constraint m, n ≤ 3 pins it at 2⁹ = 512 states — a constant for all inputs. This is the whole reason the problem flags the tiny bound: it converts an otherwise intractable "search all configurations" into a trivially fast BFS. Remove that bound and you'd need the GF(2) linear-algebra solution (§6) instead.

13   Follow-Up Bank — the interviewer's second act

Follow-up: "Before you push this to production — can you see any improvements you'd make?"
A couple. The flip-masks and the start-encoding are pure functions of the dimensions, so I'd compute them once. I'd also confirm removeFirst() isn't used — the head index keeps dequeues O(1). For a micro-win I could use a fixed-size [Bool] of length 512 as the visited marker instead of a Set, since the state space is bounded and small — array indexing beats hashing.
Follow-up: "What if the matrix could be much larger — say 30×30?"
Then BFS over 2^900 states is hopeless. This is the classic Lights Out problem; with a large grid I'd model it as a linear system over GF(2) — each flip is a vector, the target is the start matrix, and Gaussian elimination over GF(2) finds a solving combination (and its minimality) in O((m·n)³). The BFS only works because the small bound makes the state space a constant.
Follow-up: "Why is a visited set enough — couldn't a shorter path appear later?"
No — BFS explores in non-decreasing layer order, so the first time any state is reached is via the fewest flips. Once visited, any later path to it is at least as long, so skipping it is safe. That's the same first-touch guarantee that makes the goal's first arrival optimal.
Follow-up: "This is the same family as LC 773 Sliding Puzzle — what changes?"
Only the encoding and the move rule. There the vertex is a board permutation encoded as a string, and a move swaps the blank with an adjacent tile rather than XOR-ing a mask. The BFS loop — queue, visited set of encoded states, first-arrival-at-goal — is identical. Both are state-space BFS (Variant 8.3); recognizing "configuration = vertex" is the transferable skill.
