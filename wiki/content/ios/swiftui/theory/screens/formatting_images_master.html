Three small topics that share a trait: each has an obvious form that almost works, and a slightly different form that is correct. Currency formatting where the locale must be a Locale rather than a string; AsyncImage where .resizable() only exists inside the content closure; multiline text where "centred" does not look centred.

All three surface most often in list rows, where inconsistent sizing is immediately visible down a column.

THE ONE IDEA With AsyncImage, .resizable() goes inside the content closure and .frame/.clipped() go outside it. Inside, because only the loaded Image has that modifier; outside, so the sizing also applies to the placeholder — otherwise the row jumps when loading finishes.

2 Mental Model — how to think about it

AsyncImage has two distinct regions and it matters which one a modifier lands in:

AsyncImage(url:) { image in
    image
        .resizable()          <- INSIDE: applies to the loaded Image only
        .aspectRatio(...)
} placeholder: {
    ProgressView()            <- INSIDE: the loading state
}
.frame(width:height:)         <- OUTSIDE: applies to BOTH states
.clipped()                    <- OUTSIDE: crops whatever is shown
Sizing outside the closure is what keeps the layout stable across the loading-to-loaded transition.

The reason .resizable() cannot go outside is that AsyncImage is not an Image — it is a view that may eventually contain one. Only inside the closure do you hold the actual Image that has the modifier.

4 The API — types, modifiers, signatures

SwiftCurrency and numbers
// Force a specific locale for currency - pass a Locale, not a raw string.
dollarAmount.formatted(.currency(code: "USD").locale(Locale(identifier: "en_US")))

// Limit decimal places on a plain number.
dollarAmount.formatted(.number.precision(.fractionLength(2)))
ModifierWhat it controls
.aspectRatio(contentMode: .fit)Scales to stay inside the frame.
.aspectRatio(contentMode: .fill)Fills the frame and may overflow — pair with .clipped().
.frame(width:height:)Set both dimensions, not just maxWidth, so content below is predictable.
.lineLimit(_:)Caps the number of lines. Wrapping happens by default; this limits how many lines before truncating.
.truncationMode(.head/.tail/.middle)Where the ellipsis goes.
.minimumScaleFactor(_:)Shrink text instead of truncating, down to the given fraction.

5 Patterns That Work

A correctly sized remote image

SwiftClosure form, sizing outside
AsyncImage(url: viewData.imageURL) { image in
    image
        .resizable()
        .aspectRatio(contentMode: .fill)
} placeholder: {
    ProgressView()
}
.frame(width: 100, height: 100)   // applies to the placeholder too
.clipped()                          // required with .fill

Multiline text that truncates predictably

SwiftCap the lines, choose the ellipsis
Text(viewData.name)
    .lineLimit(2)
    .truncationMode(.tail)

Two fixes for crooked centred text

With .multilineTextAlignment(.center), each wrapped line centres around its own text width rather than the full block width — so a two-line title reads as visually staggered even though it is technically centred.

SwiftPick one
// Option 1: left-align titles. Most common choice for list rows.
Text(viewData.name)
    .lineLimit(2)
    .truncationMode(.tail)
    .multilineTextAlignment(.leading)

// Option 2: stay centred, but give the Text a fixed frame so every line
// centres against the same width.
Text(viewData.name)
    .lineLimit(2)
    .truncationMode(.tail)
    .multilineTextAlignment(.center)
    .frame(maxWidth: .infinity)

6 Anti-Patterns — and the fix

Calling .resizable() on AsyncImage itself

SwiftThe modifier does not exist there
// WRONG: AsyncImage(url:) alone gives a view that may eventually contain
// an Image - it has no .resizable() member of its own.
AsyncImage(url: url)
    .resizable()

// RIGHT: use the closure form so .resizable() lands on the loaded Image.
AsyncImage(url: url) { image in
    image.resizable()
} placeholder: {
    ProgressView()
}

Putting .frame inside the content closure

Sized inside, the frame applies only to the loaded image — the placeholder keeps its intrinsic size, so the row visibly resizes the moment loading completes. Outside the closure, both states share one frame.

Passing a locale identifier as a raw string

.locale expects a Locale. Handing it a bare string does not select the locale you meant, and the currency symbol quietly follows the device instead.

7 Pitfalls & Gotchas

Presentation details that bite in list rows
1. .fill without .clipped() overflows. Fill mode is allowed to exceed the frame; clipping is what cuts it back.
2. Set both frame dimensions, not just maxWidth. Otherwise the height is content-dependent and everything below shifts as images load.
3. "Centred" multiline text centres per line. Give it a fixed frame, or left-align it — the staggered look is not a rendering bug.
