Theory

You are given an array of words and an integer maxWidth. Format the words into lines of exactly maxWidth characters, fully justified. Pack as many words as fit onto each line, greedily. On each line, distribute the extra spaces as evenly as possible between words; when the spaces don't divide evenly, the left gaps get one more space than the right gaps. The last line is left-justified — single spaces between words, all remaining padding on the right. A line with a single word is also left-justified for the same reason: there are no gaps to distribute into.

InputOutputWhy
["This","is","an"], maxWidth=16["This    is    an"] (one line, 16 wide)3 words → 2 gaps; 16 − 8 = 8 spaces → 4 per gap, divides evenly
["What","must","be"], maxWidth=12["What   must  be"]12 − 8 = 4 spaces over 2 gaps → 2 each; even split
["a","b","c","d"], maxWidth=5 (as last line)["a b c d"] (left-justified, right-padded)Last line: single spaces between words, trailing pad to width

The twist is that this is pure, unforgiving bookkeeping. There is no clever algorithm to discover — the greedy packing is obvious — but the spacing has three special cases (normal interior line, last line, single-word line) and an integer-division remainder that must bias left. This problem is the #1 most-frequent Airbnb question precisely because it tests whether you can handle a thicket of edge cases without a single off-by-one.

2   Reading the Problem — insights from the constraints

"Lines of exactly maxWidth characters": every output line, without exception, is exactly maxWidth long. This is the invariant you check every case against: normal lines reach it by distributing spaces between words; last and single-word lines reach it by padding the right. If any line comes out short or long, a spacing rule is wrong.
"Pack as many words as fit, greedily": packing is not an optimization problem — there is no looking ahead for a better line break. You add words to the current line until the next one would overflow, then commit. The fit test must account for the minimum one space between every pair of words already on the line.
"Distribute extra spaces as evenly as possible, left gaps get more": with spaces total padding over gaps slots, the base per gap is spaces / gaps and the remainder is spaces % gaps. The leftmost remainder gaps each get one extra space. This left-bias is the single most error-prone integer-division detail.
"The last line is left-justified": the last line breaks the justification rule — words are separated by single spaces and the entire remaining width is padding on the right. You only know a line is the last line when you've consumed the final word, so the formatting of a committed line depends on whether more words remain.
A single-word line has zero gaps: when a line holds exactly one word, there are no inter-word gaps to distribute spaces into, so it is left-justified — the word, then right-padding. Trying to apply the even-split formula here divides by zero gaps; this case must be branched out explicitly.
Each word length ≤ maxWidth: the constraints guarantee no single word exceeds the line width, so a word always fits on a line by itself. You never need to break a word across lines — one less category of edge case.

4   Clarifying Questions

The questions to ask before you start — each answer's design consequence is noted. (No spoken plan and no code on this page: this is a TARGET.)

Clarifying questions
Q: "When the extra spaces don't divide evenly across the gaps, which gaps get the extra space — left or right?"
A: The left gaps. → base = spaces / gaps, extra = spaces % gaps; the leftmost extra gaps get base + 1, the rest get base.
Q: "How is the last line formatted differently from the interior lines?"
A: Left-justified — single spaces between words, all remaining width padded on the right. → I must know whether a committed line is the final one, so the format branch depends on remaining words.
Q: "What about a line that holds only one word — there are no gaps to fill?"
A: Left-justify it: the word followed by right padding. → A dedicated zero-gap branch; the even-split formula would divide by zero gaps.
Q: "Is every word guaranteed to be at most maxWidth long, so I never have to split a word?"
A: Yes, every word fits within maxWidth. → No word-breaking logic; a word always fits on a line by itself.
Q: "Must every output line be exactly maxWidth characters, including the last?"
A: Yes, exactly maxWidth, every line. → I verify each formatted line's length equals maxWidth as my post-condition.

The deciding signal: "pack into fixed-width lines and distribute leftover space." When a problem is obviously greedy to pack but spends its words describing spacing/padding special cases, the work is two separate passes — greedy packing into groups, then a per-group formatter with one branch per special case. Keeping the two phases apart is what makes the three spacing cases tractable.

6   Try It Yourself First

This is the hardest bookkeeping in the block. Solve it before peeking — open the hints one at a time.

Hint 1 — the constraint

Two phases, kept as two separate passes. Phase 1 decides which words go on which line (greedy packing). Phase 2 turns each committed line of words into a width-maxWidth string. Do not try to do both at once — the spacing rules are far easier to get right when packing is already done.

Hint 2 — the packing fit test

Add words to the current line while (sum of their lengths) + (one space per gap) ≤ maxWidth, where the number of gaps is (words on the line so far) if you're testing whether to add one more. The moment the next word would overflow, commit the current line as a group and start a new one with that word.

Hint 3 — the even-space distribution

For an interior line with gaps = wordsOnLine − 1 and spaces = maxWidth − totalWordLength to distribute: base = spaces / gaps and extra = spaces % gaps. The first extra gaps get base + 1 spaces; the remaining gaps get base. That left-bias is exactly what "as evenly as possible, left gaps more" means.

Hint 4 — the two special-case branches

Two lines never use the even-split: (1) the last line — join its words with single spaces, then right-pad with spaces to maxWidth; (2) any line with a single word — the word, then right-pad to maxWidth (zero gaps means nothing to distribute). Detect "last line" by whether any words remain after committing the current group.

Hint 5 — verifying yourself

Run words=["This","is","an","example","of","text","justification."], maxWidth=16. The first interior line packs "This is an" → 3 words, 2 gaps, 8 spaces → 4 each → "This····is····an". The last line "justification." is a single word → right-padded. If any line isn't exactly 16 characters, a branch is wrong.
