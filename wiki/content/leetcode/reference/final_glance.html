Final Glance

Review & Reference · Read this 30 minutes before the interview

The one-page pre-interview glance. Not a place to learn anything new — a place to reload the cross-cutting reflexes you already know but tend to forget under pressure. Each trap is a bug that has actually bitten in a solved problem; each links back to the full snippet. The strategy is rarely where you lose a problem — a forgotten reset, an off-by-one in a gap count, a divide-by-zero on the single-word case is.

THE ONE IDEA: Almost every wrong answer on a problem you understood is a small mechanical slip, not a strategy error — a forgotten state reset, an n vs n−1 separator count, an unguarded array[i] or a / b. Before you say "done", run this list against your code.

0 How to use it

Skim §1–§3 once the night before and again right before the round. During the interview, after you have a working solution, run the §1 traps against it as a final pass before declaring it done. The printable checklist at the bottom condenses everything to one screen. Every row links to its full explanation on the Code Snippets or Pattern Recognition page.

★ The printable checklist

One screen. Print it, or keep it visible. Run the "Before I say done" block against every solution.

Before I write code

  • Paraphrased the problem back; asked types / sorted / empty / invalid / size
  • Read the constraint bound → know the target complexity
  • Said the approach out loud, got agreement
  • Listed every piece of state I'll manage (and which need a shared resetState())

While I write code

  • Multi-variable state → one resetState() helper, reset in one place
  • Array(string) before indexing a String by Int
  • Separators = n − 1, not n
  • Monotonic stack: record AFTER evict, BEFORE push
  • Dictionary append default is [], not [""]
  • Greedy batch: overflowing item starts the NEXT batch
  • Narrating decisions, no duplicated logic

Before I say "done"

  • Every array[i] / array[...i] guarded against out-of-range
  • Every a / b / a % b guarded against zero divisor (single-word, single-element, empty)
  • Pivot-split: each side may be missing — handled with nil
  • Split-point loop is inclusive 0...count
  • stack.last mutation: struct? → mutate by index
  • Traced the canonical example AND a degenerate case (empty / single / last element)

After it works

  • Stated time + space with the argument, not just the label
  • Ready for "can it be improved?" — the answer is usually yes (drop a map, O(1) space, one pass)
