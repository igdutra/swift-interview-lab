# CSS vocabulary (closed set)

Single stylesheet: `wiki/_shared/wiki.css`. **Never write inline `style=""`** (validator error) and never invent a class (validator warns on classes absent from wiki.css). If a page needs a new visual, extend `wiki.css` first — tokens only via `var(--…)`, px sizes, no gradients/animations/icons/emoji. Spacing scale: 4/8/12/16/24/32/48.

## Design identity (keep intact)

Wikimedia-style: serif (`Linux Libertine O`) for h1/h2 + lead only; Source Sans 3 16px body; white content sheet on `#f8f9fa`; sticky TOC; powder-blue infobox title; Xcode Classic Light syntax tokens. One hue = one meaning: blue=info/practice, purple=insight/theory, green=tip, amber=caution, red=danger, gold=the one big idea.

## Class inventory

- **Page chrome**: `page` `content` `content-centered` (hubs) `page-title` `page-meta` `page-footer` `wiki-hr` `sec-num`
- **Nav (generated — never hand-write)**: `topnav` `nav-cat` `nav-group` `nav-trigger` `nav-caret` `nav-menu` `nav-menu-scroll` `nav-item` `current`
- **TOC (generated by toc.js — never hand-write)**: `toc-col` `toc-inner` `toc-title` `toc-cat-theory` `toc-cat-walkthrough` `toc-link` `toc-sub` `toc-num`
- **Infobox**: `infobox` `infobox-title` `infobox-row` `infobox-k` `infobox-v` (`mono`) · difficulty `tag tag-E|tag-M|tag-H`
- **Lead**: `lead`
- **Code blocks**: `pre.pseudo|swift|dry|ascii` + label row `code-label` `code-lang` `lang-pseudo|lang-swift|lang-trace|lang-ascii` `code-desc` `ascii-caption`
  - Syntax tokens: `kw fn tp st nm cm pk pc` · trace tokens: `tr-var tr-ok tr-bad tr-aha`
- **Tables**: base `table/th/td` + variants `t-decoder t-clue t-versus t-cmx` (`tc` `sc`, row `bottleneck`) `t-edge t-qmap t-example`
- **Callouts**: `co co-when co-insight co-note co-tip` · `co-idea` + `co-idea-title` (max one per page) · `co-pitfall` + `co-pitfall-title` + `pitfall` `pitfall-bullet`
- **Catalog blocks**: `pattern-block solution-block` + `block-header block-kicker block-badge block-name block-pills` `pill pill-t pill-s` `block-body` `approach-box approach-title`
- **Interview**: `sim-block sim-header sim-subheader sim-body sim-q sim-a` `speech speech-kicker` `followup followup-q followup-a` `details.hint` `solo-box solo-banner checklist`
- **Cross-links**: `ref-grid` `xref-card` (blue, "go practice") `backref-card` (purple, "go generalize") `xref-coming` (planned target — use on a `<span>`, never a dead link) + card anatomy `ref-file ref-title ref-meta ref-topics ref-desc`
- **Hub**: `hub-section-title hub-blurb hub-prose hub-pattern-title hub-pattern-note`
- **Figures**: `fig` `fig-img` `fig-img-plain`
- **Utilities (§11 — legacy absorbers; prefer semantic classes, don't grow this set)**: `u-measure u-m-0 u-mt-0/6/8/12/16/20/24/28/32 u-mb-4/16 u-my-16-12/10-6/6-6/6-0/8-0/32-0/24-8/20-16 u-pad-16-20/16-20-0/20-24 u-info-text u-note-text u-note-gap u-tip-inline u-muted-inline u-caption u-plain u-aside u-panel u-bg-caution`
